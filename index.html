Shipping Sidebar

Extensión de Chrome para relevamiento de envíos (benchmark)

Usa las flechas del teclado o los botones para navegar

Qué es Shipping Sidebar

Es una extensión de Chrome que permite gestionar tareas de relevamiento de envíos desde un panel lateral del navegador.

Cada tarea corresponde a un producto en un sitio de e-commerce. El objetivo es registrar datos como precio, vendedor, promesas de envío y costo de envío.

Los datos se almacenan automáticamente en una planilla de Google Sheets.

Qué automatiza

  • Asignación y lectura de tareas desde Google Sheets por usuario (User).
  • Apertura rápida de URL del competidor (COMP_ITEM_URL) desde el panel.
  • Formulario guiado para capturar datos clave sin ir/volver a la planilla.
  • Escritura automática en la fila correcta usando task_id como clave.
  • Medición de tiempo por tarea: Start_time, Finish_time y Time_per_task.
  • Cierre de tarea marcando is_Completed al enviar.

Por qué es buena / necesaria

  • Reduce errores: menos copiar/pegar y menos columnas equivocadas.
  • Estandariza: mismo flujo y mismos campos para todos.
  • Acelera: baja el tiempo operativo por tarea y la fricción.
  • Mejora trazabilidad: datos consistentes + tiempos reales por site/rival.
  • Escalable: el backend escribe por headers (no depende del orden de columnas).

Paso 1 — Iniciar sesión

Al abrir el panel lateral aparece un campo para ingresar el nombre de usuario.

Este nombre debe coincidir con el que figura en la planilla de tareas (User).

Después de escribirlo, hacer clic en Cargar tareas para traer las tareas asignadas.

El backend filtra: User = username y is_Completed ≠ TRUE.

Ingreso de usuario y carga de tareas asignadas.

Paso 2 — Navegar entre tareas

Se puede elegir una tarea desde el selector desplegable o avanzar y retroceder con Tarea anterior / Siguiente tarea.

Se muestra el número de tarea (task_id), el site y el CEP asignado para ese relevamiento.

El CEP viaja desde Sheets en el doGet para evitar cargarlo manualmente.

Paso 3 — Abrir la tarea

El botón Abrir tarea abre la URL del producto en la pestaña activa.

Al abrir, se registra Start_time (timestamp de inicio).

El cronómetro se reinicia cada vez que se navega a una tarea diferente.

Paso 4 — Completar el formulario

  • Precio del producto — roduct_price
  • Enviado por — Sent_By
  • Vendido por — Sold_By
  • Importado — Imported?
  • Almacenado por el rival — Stored_by_Rival?
  • Promesa de envío — shipping_promise
  • Hora límite pay-before — Deadline_to_paybefore

Formulario operativo de carga y envío de tarea.

Pegado rápido

Permite capturar texto desde la página al formulario con atajos, sin ir a la planilla.

1. Seleccionar el texto en la página

2. Presionar una tecla:

1 Precio del producto    2 Enviado por    3 Vendido por

También se puede hacer clic en los botones numerados junto a cada campo como alternativa.

Paso 5 — Enviar la tarea

Al hacer clic en Enviar tarea se envían los datos al backend junto con:

• Start_time (inicio)

• Finish_time (fin)

• Time_per_task (duración)

Al finalizar, se marca is_Completed y se avanza a la siguiente tarea pendiente.

Cómo funciona (arquitectura)

Google Sheets → fuente de verdad de tareas (cada fila = 1 tarea).

Apps Script (backend) → expone endpoints HTTP (doGet/doPost) para leer/escribir por headers.

Chrome Extension → UI lateral + navegación + envío de datos.

Clave: se escribe por nombre de columna (headers), no por índice fijo.

Stack / tecnologías

  • Chrome Extension (HTML, CSS, JavaScript)
  • Google Apps Script (Web App: doGet / doPost)
  • Google Sheets (data store operativo)
  • HTTP + JSON (Fetch/UrlFetchApp)

Consideraciones

  • El tiempo se mide por tarea con timestamps (inicio/fin) enviados desde la extensión.
  • El backend escribe en la fila correcta usando task_id como clave.
  • Si se agregan columnas nuevas, el backend puede soportarlas mientras se mantengan los headers.
  • Después de actualizar la extensión, recargar las páginas abiertas para que el pegado rápido funcione.
